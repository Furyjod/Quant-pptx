SMA Crossover Strategy

Strategy Logic

# Core Strategy Implementation
def detect_crossover(fast_ma, slow_ma):
    # Detect when fast SMA crosses above slow SMA
    above = fast_ma > slow_ma
    above_prev = above.shift(1)
    
    # Generate signals with no look-ahead bias
    long_signal = (above & ~above_prev).shift(1)
    exit_signal = (~above & above_prev).shift(1)
    
    return long_signal, exit_signal

Entry Rules

• Fast SMA (20) crosses above Slow SMA (50)

• Execute at next day's open price

• Fixed position sizing

Exit Rules

• Fast SMA crosses below Slow SMA

• No stop-loss in base implementation

• Track open positions at end

Challenges & Solutions

🔴 Challenge: NaN Handling in Moving Averages

Solution: Implemented robust NaN handling with proper boolean type casting and fillna operations to ensure signal generation integrity.

🔴 Challenge: Look-Ahead Bias Prevention

Solution: Applied systematic signal shifting to ensure decisions at time t only use data available up to t-1, maintaining realistic backtesting conditions.

🔴 Challenge: Data Availability

Solution: Created synthetic data generator using geometric Brownian motion with configurable trends and volatility for continuous testing.

🔴 Challenge: Mixed Date Type Handling

Solution: Implemented robust date parsing and formatting functions to handle various date formats and None values in trade reports.
